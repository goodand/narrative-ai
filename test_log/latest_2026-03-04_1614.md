# Search Results Round1 (2026-03-04)

## Scope
- Performance-first evidence collection for upload latency and reliability.
- Priority sources: official docs (Render, Google Gemini, FastAPI, AWS, NGINX).

## Collected Evidence (Actionable)

### 1) Render cold-start impact (free instances)
- Source: https://render.com/docs/free
- Key point:
- Free web services spin down after 15 minutes idle.
- Spin-up can take up to ~1 minute.
- Action:
- Separate cold/warm latency distributions in all benchmarks.
- Keep warm-up requests before baseline unless testing cold-start block.

### 2) Render health checks
- Source: https://render.com/docs/health-checks
- Key point:
- Render periodically probes health endpoint.
- Action:
- Use `/health` for block pre-check and incident recovery criteria.

### 3) Gemini image input constraints
- Source: https://ai.google.dev/gemini-api/docs/image-understanding
- Key points:
- Inline image data is recommended for small requests; total request size should be <20MB including prompts.
- For larger/reused images, File API is recommended.
- Supported image MIME: PNG/JPEG/WEBP/HEIC/HEIF.
- Action:
- Keep backend normalization and monitor payload sizes.
- Add request-size field in logs.

### 4) Gemini rate-limit model
- Source: https://ai.google.dev/gemini-api/docs/quota
- Key points:
- Limits are per project (not per API key).
- RPM/TPM/RPD can independently trigger errors.
- Action:
- Correlate timeout/latency spikes with quota windows and failover logs.

### 5) FastAPI multipart dependency
- Source: https://fastapi.tiangolo.com/tutorial/request-form-models/
- Key point:
- Form/file handling requires `python-multipart`.
- Action:
- Keep dependency pinned and verified in deploy logs.

### 6) ALB idle timeout and latency fields
- Source: https://docs.aws.amazon.com/elasticloadbalancing/latest/application/application-load-balancers.html
- Source: https://docs.aws.amazon.com/elasticloadbalancing/latest/application/load-balancer-access-logs.html
- Key points:
- ALB idle timeout default is 60s.
- Access logs expose request/target/response processing times.
- Action:
- If ALB used, enable log-based latency split and validate idle-timeout fit.

### 7) NGINX request buffering and timing
- Source: https://nginx.org/en/docs/http/ngx_http_proxy_module.html
- Key point:
- `proxy_request_buffering on` reads full request body before proxying.
- Action:
- For large uploads, evaluate buffering policy with caution.
