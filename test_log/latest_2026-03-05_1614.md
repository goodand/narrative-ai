# MVP Release Blockers: 20 CTO Questions (2026-03-05)

## 목적
- 현재 상태: 기능 동작은 가능하나 UX 지연으로 MVP 출시가 막힘.
- 본 문서 목표: CTO 관점 20개 질문을 **출시 게이트**로 전환.
- 출력 방식: 각 질문별 `현재 상태(확정/부분확정/미확정)` + `즉시 수집 액션`.

## 상태 정의
1. `확정`: 실측/코드/운영 설정으로 확인됨
2. `부분확정`: 일부 신호는 있으나 의사결정엔 부족
3. `미확정`: 현재 아카이브에 근거 없음

## 출시 차단 핵심 블로커 (Top 6)
1. 공식 SLO 없음 (#1)
2. 추천 알고리즘 정의 없음 (#11)
3. 추천에 Gemini 필요성 미확정 (#7/#12)
4. 트래픽/비용 모델 부재 (#15/#16)
5. 단계별 latency 분해 부재 (#5/#6)
6. 데이터/캐싱 전략 미정 (#8/#17/#18)

## 24시간 액션 플랜 (MVP unblock)
1. Product/SLO 결정 회의 (60분)
- 산출물: `MVP SLO v1` (`p95`, `timeout_rate`, `cost/request`)
2. 파이프라인 정의 확정 (90분)
- 산출물: UX 단계도 + 추천 알고리즘 책임분리(로컬/서버/Gemini)
3. 계측 보강 (반나절)
- 단계별 timer + request size + cold/warm + concurrency level
4. 비용/트래픽 모델 작성 (90분)
- DAU 시나리오 3개, Gemini 비용 추정
5. 캐시/프리컴퓨팅 결정 (60분)
- daily cache 여부/TTL/무효화 규칙

## 출시 게이트 (Go/No-Go)
