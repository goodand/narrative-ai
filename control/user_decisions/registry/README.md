# User Decisions Registry

`registry/` under `user_decisions/` is for synchronization around current user-choice state.
It is not the place for the decision documents themselves.

## Good Fits
- current-decision indices
- open/closed question status pointers
- machine-readable links to the active canonical decision documents

## Not For
- ADR text
- notes
- reports
- attachments

Those belong in `resources/`.
